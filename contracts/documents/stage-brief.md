# Document Contract: `stage-brief.md`

## Purpose

Provide the runtime with one compact Markdown brief that names the stage, the goal, the key inputs, the target outputs, and any special constraints.

## Required sections

- `Stage`
- `Goal`
- `Inputs`
- `Outputs`
- `Constraints`
- `Open questions`

## Validation cues

- the file references the active stage correctly,
- the input and output document paths are explicit,
- the brief does not invent missing user answers.
