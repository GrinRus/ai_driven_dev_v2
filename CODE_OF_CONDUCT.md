# Code of Conduct

## Our standards

Project spaces should stay focused, respectful, and useful for maintainers and contributors.

Expected behavior:

- use clear and professional language;
- assume technical discussions need evidence;
- keep issue reports reproducible and scoped;
- respect privacy and do not post secrets, private logs, or private repository contents.

Unacceptable behavior:

- harassment, threats, or personal attacks;
- publishing private information without explicit permission;
- knowingly posting credentials, tokens, or sensitive runtime logs;
- derailing project spaces with unrelated disputes.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; restrict participation; or
escalate to GitHub moderation when behavior harms the project or its users.

Report conduct concerns privately through the repository owner profile. Do not include
sensitive security details in conduct reports; use `SECURITY.md` for vulnerabilities.
