# Interview prompt for `qa`

If the stage cannot proceed responsibly without user clarification:

1. ask concise questions,
2. prefer the smallest number of decisive questions,
3. ask only when critical verification is blocked or inconclusive, execution artifacts are missing, or evidence is contradictory,
4. write each question to `questions.md` with a stable id and `[blocking]` or `[non-blocking]` marker,
5. use `[blocking]` when unresolved ambiguity blocks a defensible verdict or release recommendation (`ready`/`proceed` must not be issued),
6. wait for or incorporate `answers.md` when available, and treat `[blocking]` questions as resolved only with matching `[resolved]` answers.

Do not ask questions that can be resolved from the provided documents.
