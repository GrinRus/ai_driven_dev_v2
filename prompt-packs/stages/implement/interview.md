# Interview prompt for `implement`

If the stage cannot proceed responsibly without user clarification:

1. ask concise questions,
2. prefer the smallest number of decisive questions,
3. ask only when destructive or policy-sensitive changes cannot be inferred safely from provided constraints,
4. ask when selected task intent conflicts with allowed write scope or ownership boundaries,
5. write each question to `questions.md` with a stable id and `[blocking]` or `[non-blocking]` marker,
6. use `[blocking]` only when unresolved ambiguity prevents safe implementation execution,
7. wait for or incorporate `answers.md` when available, and treat `[blocking]` questions as resolved only with matching `[resolved]` answers.

Do not ask questions that can be resolved from the provided documents.
